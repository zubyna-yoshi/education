# 산업안전보건기준 조항-현장 적용 시스템(MVP)

## 1) 해결하려는 문제
- 같은 현장 상황도 감독관 경력/전문분야에 따라 적용 조항이 달라질 수 있음
- 법령 텍스트만으로는 현장 사진/행위와 조항을 빠르게 매칭하기 어려움
- 위반인지 여부, 행정조치/형사절차 분기 기준이 개인 경험에 의존됨

## 2) 핵심 아이디어
각 조항마다 `조항 적용 카드`를 만들고, 감독관이 현장 사진/영상/메모를 카드에 연결하여 누적한다.

카드에는 아래를 고정 필드로 둔다.
- 조문 식별: 편/장/절/조/항/호
- 현장 징후: 사진에서 보이는 위험요인 태그
- 적용 판단: 성립요건/제외요건/판단 난점
- 조치 가이드: 즉시 시정, 작업중지, 사법처리 검토 기준
- 증거 가이드: 어떤 사진/문서/진술이 핵심인지

## 3) 사용자 역할
- 근로감독관: 사건 등록, 조항 매핑, 증거 업로드, 조치기록
- 팀장/검토자: 조항 매핑 승인, 판정 일관성 검토
- 관리자: 조항 마스터 관리, 템플릿/태그 관리, 통계

## 4) 화면/기능(MVP)
### A. 조항 검색
- 조문번호/키워드/위험유형(추락, 협착, 감전 등)으로 검색
- 검색 결과에 `대표사진 + 자주 발생하는 위반패턴` 표시

### B. 사건 등록
- 현장 정보: 업종, 공정, 설비, 작업유형, 피해유형
- 자료 업로드: 사진/동영상/문서
- AI 보조(선택): 사진 위험요소 후보 태깅(최종 판단은 감독관)

### C. 조항 적용 보드
- 좌측: 사건 증거
- 우측: 후보 조항 카드(근거 체크리스트 포함)
- 카드별로 `해당/불해당/추가증거필요` 판정

### D. 조치 추천(의사결정 보조)
- 즉시 위험이면 작업중지/긴급개선 우선 안내
- 위반 성립 시 행정조치 + 형사검토 체크리스트 표시
- 자동 생성: 현장조치 메모 초안, 추가확보 증거 목록

### E. 이력/학습
- 유사사례 추천(같은 공정/위험유형/조문)
- 팀 단위 판정 불일치 사례 리포트(교육 데이터)

## 5) 데이터 모델(핵심)
### `law_clause` (조항 마스터)
- id
- law_name (예: 산업안전보건기준에 관한 규칙)
- article_no (예: 제38조)
- paragraph_no, subparagraph_no
- title
- legal_text
- plain_guide_text (쉬운 설명)
- hazard_tags[]
- effective_date

### `clause_card` (조항 적용 카드)
- id
- law_clause_id
- applicability_conditions[] (성립요건)
- exclusion_conditions[] (제외요건)
- evidence_required[]
- immediate_actions[]
- admin_actions[]
- criminal_review_points[]
- canonical_examples[] (대표 사례)
- review_status (draft/reviewed/approved)

### `case_file` (사건)
- id
- site_type, industry, process_type
- incident_date, inspector_id
- risk_level (low/medium/high/critical)
- summary

### `case_evidence`
- id, case_file_id
- file_url, file_type
- captured_at
- evidence_tags[]
- location_note

### `case_clause_mapping`
- id, case_file_id, law_clause_id
- decision (applied/not_applied/needs_more)
- rationale
- reviewer_comment

## 6) 형사(범죄) 검토 분기 템플릿
주의: 실제 사건의 최종 법률 판단은 수사/사법 절차에서 확정되므로, 시스템은 `판단 보조`로 설계.

1. 위험의 중대성
- 사망/중상/동시다발 위험 여부
- 즉시 조치 미이행 여부

2. 의무 주체 특정
- 사업주/관리감독자/도급관계상 책임주체 식별 가능성

3. 위반과 결과의 인과관계 자료
- 현장사진, 설비상태, 작업지시, 교육기록, 점검기록

4. 반복/고의/인지 가능성
- 이전 지적사항 재발 여부
- 위험 인지 후 방치 정황

5. 조치 경로
- 행정조치 우선 + 형사검토 필요 플래그
- 추가 확보 증거 자동 체크리스트 출력

## 7) 바로 시작하는 구축 순서(4주)
1주차
- 조항 마스터 50~100개 우선 입력(사고다발 조항)
- 태그체계 확정(업종/공정/위험유형)

2주차
- 사건 등록 + 증거 업로드 + 조항 검색 구현
- 조항 적용 카드 편집/승인 워크플로 구현

3주차
- 의사결정 보조(해당/불해당/추가증거) + 조치 가이드 연결
- 유사사례 추천(규칙기반) 추가

4주차
- 시범운영(소규모 팀)
- 판정 일치율, 처리시간, 재작업률 측정

## 8) 운영 원칙
- 시스템은 판단 보조이며 최종 책임 판단을 자동화하지 않음
- 모든 판정에 `근거 증거`와 `판단자`를 남김
- 버전관리: 법령 개정일 기준으로 카드 버전 분리

## 9) 다음 입력 자료(사용자 제공 시 정확도 급상승)
- 자주 적용하는 상위 30개 조항 목록
- 현장에서 자주 혼동되는 유사 조항 쌍
- 실제 사진 20~50건(개인정보 비식별)
- 팀 내부 보고서/처분서식(가능한 범위)
